Grupo 1 — Expediente digital (Wix + Supabase)

Registro (Capa 0)
BENALICONPRO
▼
~10 camposRegistroWix + Supabase

Punto de entrada de todos los perfiles. Captura lo mínimo para crear la identidad digital. Lo diligencia el propio usuario externo.

Campos: nombre o razón social, tipo documento (CC/CE/NIT/Pasaporte/PEP/PPT/TI), número documento, email, teléfono, tipo de perfil.

Al completar: crea registro en Supabase (profiles), carpeta en Drive, item en Monday. Dispara consentimiento C1.

Estado: por construir en Wix.

Ubicación (Capa 1)
BENALICONPRO
▼
~18 camposPost-registroWix + Supabase

Datos de origen, migración y residencia. Mismo formulario base para todos; jurídicas agregan dirección de sede y sedes adicionales. Lo diligencia el propio perfil como tarea asignada.

Campos: país/departamento nacimiento, nacionalidades, doble nacionalidad, estatus migratorio, país/departamento/municipio residencia, zona (urbana/rural/rural dispersa), tiempo residencia, víctima desplazamiento, estrato socioeconómico. Jurídicas: dirección sede principal, sedes adicionales.

Precarga: nombre y documento desde Capa 0.

Estado: campos definidos en mockup. Plataforma: Wix + Supabase (decisión tomada).

KYC Persona Natural (Capa 2)
BENCON
▼
~25 camposPost-registroWix + Supabase

Compliance para personas naturales. El perfil lo diligencia, el analista revisa y aprueba/rechaza cada documento.

Declaración AML: actividad económica, ingresos mensuales, origen de fondos, declaración de licitud.

Declaración PEP: ¿es PEP o tiene vínculos? Si sí: cargo, entidad, fecha inicio/fin, vínculo. Repetible para múltiples PEP.

Documentos: cédula frente, cédula reverso, selfie. Contratistas agregan: certificación bancaria (<30 días).

Consentimiento previo: C2 (identificación) obligatorio antes de iniciar.

Nivel de diligencia: Simplificada para beneficiarios. Normal para contratistas.

Estado: campos definidos en mockup KYC. Plataforma: Wix + Supabase (decisión tomada).

KYC Persona Jurídica (Capa 2)
ALIPRO
▼
~35 camposPost-registroWix + Supabase

Compliance reforzada para personas jurídicas. Lo diligencia el representante legal. Analista revisa.

Declaración AML corporativa: ingresos anuales, patrimonio, origen de fondos.

Declaración PEP: del representante legal. Repetible.

UBO (beneficiarios finales >25%): nombre, documento, nacionalidad, % participación de cada uno.

Documentos: RUT, Cámara de Comercio, NIT, cédula rep. legal, estados financieros (último año fiscal), pólizas (si aplica), certificación bancaria.

Consentimiento previo: C2 (identificación) + Política P01 (SICE).

Nivel de diligencia: Reforzada.

Estado: campos definidos en mockup KYC. Plataforma: Wix + Supabase (decisión tomada).

Consentimientos C1–C7
BENALICONPRO
▼
7 ítems de firmaMomento variableWix + Supabase + Firma-GAS

Cada consentimiento se pide justo antes de necesitar los datos que autoriza. No se piden todos juntos.

  • C1 — Datos básicos → al registrarse (todos)
  • C2 — Identificación → antes de KYC (todos)
  • C3 — Datos sensibles (discapacidad, salud, género, etnia) → antes de caracterización (solo beneficiario)
  • C4 — Analítica e IA → antes de módulos QoL/DEI (beneficiario + aliado)
  • C5 — Comunicaciones → al registrarse, opcional (todos)
  • C6 — Imagen, voz, testimonios → antes de intervención (solo beneficiario)
  • C7 — Seguimiento del proceso → antes de intervención (solo beneficiario)

Firma-GAS registra: persona_id, código consentimiento, versión del texto, timestamp, IP, user-agent. Cumple Ley 527/1999 y Ley 1581/2012.

QoL Autoinforme (INICO-FEAPS)
BEN
▼
~46 campos LikertOperaciónGoogle Forms

Escala INICO-FEAPS estandarizada, versión autoinforme. La responde el propio beneficiario (con apoyo si necesita). Mide 8 dimensiones: autodeterminación, derechos, bienestar emocional, inclusión social, relaciones interpersonales, bienestar material, desarrollo personal, bienestar físico. No se separa: instrumento estandarizado validado académicamente. Separarlo rompería integridad y complicaría cálculos de dimensiones en GAS.

Alimenta: módulo QoL GAS → Gemini → reporte multi-agente.

Consentimiento previo: C4 (analítica/IA).

Estado: en producción.

Registra: persona_id, código consentimiento, versión texto, timestamp, IP, user-agent. Cumple Ley 527/1999 y Ley 1581/2012.

Posibles adicionales

  • SARLAFT — origen lícito de fondos (KYC)
  • PEP — persona expuesta políticamente (KYC)
  • NDA — confidencialidad (contratistas, aliados)
  • Contrato vinculación — términos por perfil

Criterio de cuándo pedir

Justo antes de necesitar los datos que autoriza. No todo junto al registrarse — reduce fatiga de checkbox y mejora calidad del consentimiento.
